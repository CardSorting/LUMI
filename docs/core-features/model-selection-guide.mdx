This guide reflects the **providers wired in this build**. The settings dropdown reads `src/shared/providers/providers.json`; routing is implemented in `src/core/api/index.ts`.

## Supported providers

| Provider key | UI label | Handler |
|--------------|----------|---------|
| `openrouter` | OpenRouter | `OpenRouterHandler` (default fallback) |
| `openai-codex` | ChatGPT Subscription | `OpenAiCodexHandler` |
| `nousResearch` | NousResearch | `NousResearchHandler` |
| `cloudflare` | Cloudflare Workers AI | `CloudflareHandler` |

If an unknown provider is selected, `buildApiHandler` falls back to OpenRouter.

## Plan vs Act

LUMI can use **different providers per mode**:

- **Plan mode** — `planModeApiProvider` + plan model IDs
- **Act mode** — `actModeApiProvider` + act model IDs

Configure both in the LUMI sidebar **Settings** panel.

## How to select a model

<Steps>
  <Step title="Open LUMI Settings">
    Click the gear icon in the LUMI sidebar, or run **LUMI: Settings** from the Command Palette.
  </Step>
  <Step title="Choose provider">
    Select one of the four providers for Plan and/or Act mode.
  </Step>
  <Step title="Add credentials">
    Enter the API key or complete OAuth for that provider (see [authorizing](/getting-started/authorizing-with-dietcode)).
  </Step>
  <Step title="Pick a model">
    Choose model ID from the provider's picker. OpenRouter exposes the widest catalog.
  </Step>
</Steps>

## Provider notes

### OpenRouter (recommended default)

- Access to many frontier and open models through one API key.
- Supports reasoning/thinking budget settings when the model allows.
- Used as fallback when provider routing fails.

### ChatGPT Subscription (`openai-codex`)

- For users with an OpenAI Codex / ChatGPT subscription flow.
- No separate OpenAI API key billing path.

### NousResearch

- Direct access to NousResearch-hosted models.
- Requires a NousResearch API key.

### Cloudflare Workers AI

- Requires Cloudflare account ID and API token.
- Runs inference on Cloudflare's platform.

## Additional provider files

The repository contains handler implementations for other backends under `src/core/api/providers/` (Anthropic, Ollama, Bedrock, etc.) and matching settings components under `webview-ui/src/components/settings/providers/`. Those handlers are **not registered** in `buildApiHandler` in this build — only the four providers above are active.

## Context window

Large tasks may require models with bigger context windows. LUMI tracks usage in `src/core/context/` and offers `/compact` and `/smol` slash commands to reduce history size.

See [context windows](/model-config/context-windows) for general guidance.
