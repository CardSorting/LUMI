LUMI is an AI coding agent that runs as a VS Code extension (`CardSorting.lumi`). It gives you direct access to frontier models through supported providers, with approval gates on every file change and shell command.

  </Step>

## What LUMI can do

With your permission, LUMI can:

- **Read and write files** — Navigate the workspace, create files, and apply targeted edits. Changes appear in the diff view before you approve.
- **Run terminal commands** — Execute tests, install packages, and debug with live output.
- **Use a browser** — Launch Puppeteer to test web apps and capture screenshots.
- **Connect external tools** — Extend capabilities with [MCP servers](/mcp/mcp-overview).
- **Delegate to subagents** — Spawn background agents for parallel research or verification ([subagents](/features/subagents)).
- **Steer with ROADMAP.md** — Optional auto-rolling roadmap gates (`lumi.roadmap.*` settings).

## Workspace integration

LUMI operates directly on your local project:

- **Local files** — Reads and writes in your workspace; respects `.dietcodeignore`.
- **Local terminal** — Commands run in your environment with your installed tools.
- **Local persistence** — Task history and cognitive memory use BroccoliDB (SQLite) on your machine via `@noorm/broccolidb`.
- **Human-in-the-loop** — Mutating tools require approval unless auto-approve rules allow them.

## Supported model providers

This build wires four providers in `src/core/api/index.ts` (see `src/shared/providers/providers.json`):

| Provider | Use case |
|----------|----------|
| **OpenRouter** | Default; broad model catalog with your API key |
| **ChatGPT Subscription** | OpenAI Codex subscription flow |
| **NousResearch** | NousResearch-hosted models |
| **Cloudflare Workers AI** | Cloudflare account + API token |

Plan and Act modes can each use a different provider.

## Internal naming

User-facing copy says **LUMI**. Code and config may still reference **LUMI** (message types, `.dietcoderules/`, storage keys). That is expected — it is the internal module prefix, not a separate product.

## Why LUMI

- **Agency with control** — Terminal, browser, and file tools with explicit approval.
- **Local-first context** — BroccoliDB stores task history and memory on disk.
- **Provider choice** — Switch models without leaving the sidebar.
- **Extensibility** — MCP, hooks, skills, workflows, and subagents.
